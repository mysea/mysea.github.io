发表于 2018-04-08 | 分类于 Java EE | 本文总阅读量 次

在Java EE框架中，一开始是SSH(Spring+Struts+Hibernate)，后来MyBatis替代了Hibernate，成为SSM(Spring+Struts+Mybatis)，再后来Struts又被Spring MVC替代了，现在Spring Boot又替代Spring MVC了。

Spring有两大特点：IOC和AOP，分别是控制反转和面向切面编程。IOC意思是不再用new创建对象，而是由Spring去创建和管理；AOP指将核心业务和周边业务代码分离开来，利用切面将两者“编织”在一起。

AOP

没有AOP的时候核心业务比如数据库操作，周边业务比如日志输出，性能统计等是写在一块的，这样不利于代码的维护和扩展，采用AOP之后，核心业务和周边业务的代码分开来写，利用切面将两者“编织”在一起，在执行核心业务的前后执行周边业务，既实现了周边业务，又达到代码分离的效果。
