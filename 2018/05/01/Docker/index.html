Docker

发表于 2018-05-01 | 本文总阅读量 次

Docker（集装箱），确实有点像轻量级的虚拟机，但又不相同。有了Docker，在一台物理机器上可以运行多个容器，每个容器可以提供不同的服务，容器是由镜像生成的，而镜像，既可以用Dockerfile构建，也可以直接从官方仓库(DokcerHub)或者私有仓库上pull下来，为了方便地管理多个容器，docker还提供了DockerCompose，将多个容器的配置信息写在yml文件中，一次可以启动多个容器。有了docker，环境是可以共用的，可以将镜像推送到仓库上，其他人可以下载然后生成容器，这样就保证了不会出现“在我的机器上是可以运行的”这种情况，并且docker镜像或容器的创建删除都很方便。

Dockerfile

Dockerfile是自动构建docker镜像的配置文件，Dockerfile中的命令非常类似linux shell下的命令

Dockerfile可以让用户自定义构建docker镜像，支持以 # 开头的注释行

通常包含四部分：

  • 基础镜像(父)镜像
  • 维护者信息
  • 镜像操作命令
  • 容器启动命令

ENTRYPOINT & CMD

ENTRYPOINT

使用方法：

  1. ENTRYPOINT [“executable”, “param1”, “param2”] (the preferred exec form)

    推荐使用1方法，启动起来后，pid为1

  2. ENTRYPOINT command param1 param2 (shell form)

    启动起来后，pid号为shell命令执行完的pid号

CMD

使用方法：

  1. CMD [“executable”,”param1”,”param2”] (exec form, this is the preferred form)
    运行一个可执行的文件并提供参数

  2. CMD [“param1”,”param2”] (as default parameters to ENTRYPOINT)
    为ENTRYPOINT指定参数

  3. CMD command param1 param2 (shell form)
    以”/bin/sh -c”的方法执行的命令

[]内参数解释如下

-it 容器的 Shell 映射到当前的 Shell，然后你在本机窗口输入的命令，就会传入容器。
-d 启动docker容器在后台，执行命令后返回容器ID
-p 22 将宿主机的一个未使用的随机端口映射到容器的22端口，每次宿主机或容器重启后，映射端口改变
-P 2222:22 容器的 22 端口映射到本机的 2222 端口，宿主机或容器重启后，映射端口不变
–name 容器名 给容器命名
–rm 容器停止运行后，自动删除容器文件
-v host_dir:container_dir 同步宿主机host_dir目录下和容器container_dir目录下的文件
-e name=value 向容器传入环境变量
–link containName 连接已存在的容器

Jenkins持续集成

在上面的操作中，我们以命令行的方式构建镜像，再由镜像构建容器，再将镜像推送到registry上，当容器或镜像数量很多的时候，每次修改代码需要重新部署时执行这些操作是极其繁琐的，Jenkins就是一个可以自动集成的工具。

下载Jenkins镜像，生成Jenkins容器后，通过配置github，配置Jenkins自动构建的脚本，当修改代码后，直接将代码上传到github上，Jenkins可以自动检测到代码发生了变化，然后会去github下载相应的Dockfile文件，重新生成镜像，生成容器，将新的镜像推送至registry上。当第一次配置之后，以后修改便无需手动的用命令行的方式去build或run。
