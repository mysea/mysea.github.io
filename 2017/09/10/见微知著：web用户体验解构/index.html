见微知著：web用户体验解构

发表于 2017-09-10 | 分类于 读书笔记 | 本文总阅读量 次

从标题可以看出这本书的内容是关于web的用户体验，书不厚，更像是一本使用说明书。作者从4个方面来解构web用户体验，从整体到局部，从总体印象到细节，分别是界面布局和配色，主要功能区域，页面元素，其他(文字图片)。

书中的内容存在一定滞后性，比如现在很多表单输入框旁边已没有标签，取而代之的是输入框中的提示文字，验证码很多已是用手机短信验证码，内容较多的网站不再采用分页而是异步加载的方式显示下一页的内容等，而书中的内容更偏向于过去的门户网站、电子商务网站。

界面布局和配色

界面布局

按照分栏方式不同，可以分为一栏式，二栏式，三栏式，采用哪种取决于信息量和页面类型，并且根据信息的主次选择合适的比例。

一栏式

常见于企业网站、小型商务网站等。

二栏式

最常见的布局方式，分为左窄右宽、左宽右窄和左右均等(不常见)。

三栏式

又分为中间宽两边窄，两栏宽一栏窄等，常见于门户网站。

配色

取决于目标用户群体、网站类型、品牌形象等。

主要功能区域

头部

指的是网站的顶部区域，通常包含Logo、Slogan、Banner、导航等。主要分两种：实用型、设计型，实用型一般只包含Logo、导航条、快速入口等，例如大型知名网站；设计型更多体现在个性化网站。

头部元素主要包括Logo、Slogan，Logo代表品牌形象，基本作用是单击之后可以返回首页，Slogan要求琅琅上口，体现网站的特色。

尾部

通常放置联系方式、版权信息等，除此之外，也可以放置站点地图或辅助导航。

导航区

这里的导航主要指全局导航、辅助导航。全局导航指网站每个页面上始终保持同一位置的导航元素；辅助导航指全局导航下的分类筛选。

横向导航

树状导航

这个常见了，尤其是在电商网站

树状导航要注意的是：

  1. 排序规律，将分类按照一定的规律排序；
  2. 导航位置，如果想让用户关注导航，就将其放在页面左侧，如果想让用户关注内容，就将其放在右侧(强迫症的我表示受不了: )

辅助导航

面包屑导航

还挺好用的，在同一页面不同级别之间跳转的时候，不过忘了是在哪里看到的了，只能找了这个

适用条件：

层级较深；网站内容不存在交叉。

网站地图

一般在网站底部

推荐式导航

就是猜你喜欢，为你推荐，买了这个的人还买了，85%的人还看了…..

引导性导航

告诉你完成了哪些，正在进行的，剩余还未进行的，还是很友好的一种提示方式。

搜索区

主要分为搜索引擎搜索区、网站内的全局搜索、站内局部搜索

登录注册区

注册的设计要点

  1. 告诉用户注册的好处

  2. 用邮箱取代用户名作为唯一身份识别

  3. “记住登录状态”和“忘记密码”的细节
  4. 不必要情况尽量少使用验证码
  5. 优化注册表单（内容不宜过长/最好分类/及时提示/及时反馈）
  6. 友好的提示信息
  7. 设计醒目的登录注册按钮

内容展示区

用户浏览网站的不同阶段可以分为海选阶段>>对比阶段>>决策选择阶段，对应的是首页>>列表页>>详情页。

首页

内容选择方面，一般是精心挑选的，现在都是个性化推荐了……

内容呈现方式，简洁型：首页只提供入口，起到分流作用；丰富型：几乎包含了网站所有内容，页面很长很长(现在几乎没有了吧？除了下拉自动加载那种)。一般来说，重要的内容采用具有冲击力的图片，大字号，颜色鲜艳等突出显示；略次要内容采用图文结合的方式；普通内容用文字链接即可。

页面数量，取决于网站类型，如果是资讯类，长点也正常。

列表页

作为首页和详情页的过渡，信息比首页更聚焦，用户可以根据标题、图片、相关参数进行对比，从中选择感兴趣的页面进入详情页。

详情页

在用户浏览完详情页之后，就可能完成目标，离开网站，此时可以通过相关推荐，热门内容，排行榜等吸引用户浏览更多内容。

广告区

放置在网站头部的Banner广告，面积较小的按钮式广告，讨人厌的弹窗广告，会随着鼠标上下移动“跟屁虫”一样的浮动式广告，显示完一会就消失的动态广告，鼠标放上去会放缩的伸缩式广告，文字链接广告，其他比如视频广告等。

设计要点

  1. 选择合适的展现形式

  2. 根据浏览习惯个性化推荐广告以提高点击率

  3. 广告不应隔断或削弱相关内容之间的联系
  4. 避免过多广告同时出现
  5. 规范统一广告的尺寸

页面元素

表单

表单布局的几种方式

设计要点

  1. 对表单进行逻辑分组；
  2. 填写引导；
  3. 取消不必要的表单项；
  4. 标明必选可选；
  5. 最好有默认选项，比如地区等；
  6. 文字易于理解，给予提示和帮助；
  7. 操作完后给出反馈等等。

例如

网易新闻的注册表单，可以看出：

表单采用右对齐的方式；必选项用*表示；输入框下方是灰色的提示性文字，点击对应的输入框，文字会变成黑色；邮件地址后缀默认给出；如果未输入，会用红色显示输入框，在下方会给出红色提示文字；验证码看不清楚可以更换；注册按钮比较显眼

模块标签

什么是模块标签

在页面内的某个模块的内容上方使用一系列标签形式的链接来进行切换，部分解决有限网页种放置更多内容的问题

使用模块标签的条件

  1. 有多窗格的内容，但空间有限，一次只能显示一个窗格的内容

  2. 不同窗格内容相互独立，不存在前后文关系

  3. 切换内容时不需要链接到其他网页

  4. 有2-10个类别

  5. 标题短

  6. 需要突出当前正在查看的内容窗口

设计要点

  1. 突出选中的标签；
  2. 标签的跳转可以使鼠标移过自动跳转(为避免误操作的情况，会有0.1s的延迟)，也可以点击标签进行跳转；
  3. 对于同一产品相同内容的操作应是一致的；
  4. 重要的、常用的标签放在前面。

例如

网易新闻的模块标签，总共有8个标签，点击更多会出现其他未显示标签；第一个标签是要闻，其次是定位到你所在城市的新闻；鼠标移动到对应标签，背景色变为红色，字体加粗，颜色变为白色；鼠标移动之后，延迟一会儿，在下方内容出现对应的新闻，不需要刷新或跳转网页

但是可以注意到标签和内容之间的联系并不是很明显，标签上部的红色线条移到下面来更合适，比如向下面这样，标签背景色和内容背景色一致：

翻页

当页面过多时需要将内容翻页，通常包含第一页、最后一页、上一页、下一页，还可能包含具体页码、总页码、页面间距、快速跳转某一页。

链接

链接指页面之间的跳转，包括文字链接/图片链接。

设计要点

  1. 链接颜色；
  2. 可点击范围；
  3. 链接的下划线；
  4. 链接点击前后颜色变化；
  5. 区分外部链接/内部链接；
  6. 链接的交互方式(在当前页面打开还是打开一个新页面)。

按钮

如何增加按钮点击率

  1. 使用饱和度高的颜色

  2. 文字和按钮背景的颜色对比要强烈

  3. 文字简洁，以动词为主
  4. 大尺寸

设计要点

  1. 按钮的主次关系；
  2. 摆放位置；
  3. 减少用户定位时间；
  4. 优化视觉效果（采用圆角，浮出感，饱和度高，字体和背景颜色对比大）

例如

淘宝网某一商品详情页面：

按照从左往右看的习惯，立即购买放到左边是比较合理的，因为加入购物车用户还有思考的时间。采用圆角看起来没那么方正，但是，立即购买才应该用橙色强调才对吧，相比加入购物车，更希望的是用户能够立即下单。

操作反馈

当用户在网站上进行交互的时候，网站应该对用户的输入做出反应或者输出，让用户能够了解当前操作的结果和状态。

反馈的种类

成功

给出文字提示或者弹出对话框。

失败

如果失败，最好能告诉用户哪里失败了，如果是填写表单，最好不要将之前的用户填写的内容清空。

再次确认

对于某些操作，比如删除，应给予二次确认的机会，甚至可以提供撤销选择。

进度条

对于耗时操作，给出进度条，显示完成进度/剩余处理时间等

验证码

验证码的存在是为了避免用户短时间内的重复尝试，比如防止黑客破解密码，爬虫爬取网站等。

设计要点

  1. 不要滥用验证码，最好是多次失败后才使用验证码

  2. 对于很难辨认的情况，用户可以重新选择一张新的图片，或者减少生僻字、易混淆的情况出现的几率

  3. 及时验证，及时反馈，比如用户在浏览器输入完即显示是否输入正确
  4. 减少切换输入法

其他

文字设计

字体类型：手写字/艺术字，有衬线/无衬线(有衬线比如宋体，适合印刷；无衬线比如微软雅黑，适合电脑上显示)

字号

em，ex，px，百分比等，中文网站采用12px，14px字号为宜。

除此之外，图片质感要好，文字语气要谦逊，文字标题要吸引人(标题党么？)

码字不易，排版更难
