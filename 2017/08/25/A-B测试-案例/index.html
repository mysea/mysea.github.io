A/B测试--案例

发表于 2017-08-25 | 分类于 产品 | 本文总阅读量 次

顾名思义，A/B测试，一个为实验组，一个为对照组，通过比较来测试新产品和新功能。

案例概述

在这个例子中，假设有一个名为Audacity的在线教育网站，专攻在线金融课程。一般新用户访问网站都会经历以下过程：访问首页->浏览页面->创建账户->完成(注册/购买课程等)，在这个过程中第一步的人是最多的，越到后面的人数越少。会呈现出一个漏斗的形状，也就是客户漏斗模型。

值得注意的是，用户不总是按照这个漏斗从上往下走，有可能在几个过程之间来回跳转，而老用户则会直接跳过这些过程。

在Audacity网站的首页有个 start now 的按钮，点击这个按钮会出现Audacity的课程列表。现在我们将这个按钮从原来的橙色改为粉红色，来测试会增加多少学生访问Audacity的课程，也就是漏斗模型的第二层。

度量选择

将按钮颜色从橙色改成粉红色之后，我们用什么来度量改变后的影响呢？下面有四个选项：

  1. 完成课程的总人数
  2. 按钮点击数
  3. 按钮点击比率
  4. 按钮点击概率

选项1时间太长，学生完成一门课可能需要两个月；选项2计算的是数量，相比数量，百分比更合适；选项3是指 点击数/页面浏览数；选项4是指 独立点击数/独立页面浏览数，比如说有两个人先后访问后网站首页，其中第一个人点击了0次(没有点击)就走了，第二个人点击了5次(因页面加载缓慢而多次刷新的情况)，按照选项3的算法就是 5/2=2.5，而按照选项4的算法就是 1/2=0.5。

其实这个就好比是PV和UV的区别，UV的话同一个用户访问了5次但仍然只算1次，而PV的话则算5次。

如果我们想测量可用性，就采用点击率，因为点击率可以告诉我们用户多经常的找到这个按钮；而当我们想要测量总影响，比如说进入网站的二级页面，就不要把点击多次/刷新的情况考虑进去，就采用点击概率。所以在这里我们选择按钮点击概率来作为度量指标。

重复实验

现在我们做了一个实验 访客数=1000，独立点击数=100，点击概率=10%。假如我们重复实验，得出不一样的结果，比如独立点击数=100/101/110/150/900，这时候又该怎么办呢？你怎么知道哪些结果是可信的？我们在做实验统计的数据中，一般都会符合某些分布，在这个案例中，如果将点击视为成功，没有点击视为失败，那么将会有两种结果，这就符合二项分布了。
