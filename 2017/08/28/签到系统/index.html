签到系统--过程回顾

发表于 2017-08-28 | 分类于 个人作品 | 本文总阅读量 次

当时做的时候没有做详细的记录，现在过去一段时间了，也忘了好些，过程大概是这样的…..

起因

签到系统是软件工程课上老师布置的一个期末大作业，要求开发一款扫描二维码签到的安卓app，用来上课点名签到用，要求二维码可以设置失效时间，签到表可以导出为excel。

当时我的反应就是教师用网站生成二维码，然后学生用安卓app扫描二维码签到。

可问题是二维码怎么办？哪里来的二维码？用手机扫描二维码登录网站的情景我用微信qq登录网站时用过，二维码失效我也只在微信二维码上看到过，可问题是我怎么做出微信扫描二维码登录的效果？？？

转机

苦思冥想之后还是不知道该怎么办，于是就开始在网上搜啊搜，结果找到一个视频，虽然它跟签到没关系，而且我也没看太懂它说的，关键是给了我思路！

它给我的思路就是二维码其实只是信息的另一种表示，比如我用字符串生成一个二维码，其实这个二维码扫描得到的结果就是那一串字符而已，那么我在网站上利用第三方的生成二维码的接口，传递一个字符串生成一个二维码，然后手机扫描这个二维码，得到对应的字符串，再携带着这个信息访问网站的api接口，就可以实现签到了!

这个生成字符串的二维码是什么呢？我的初始设想是教师要填写课程名，有效时间，然后利用这些信息来生成这个二维码。

事实证明，我想多了……

行动！画原型！

根据我以上的设想，开始拿出草稿纸，先画网站，比如要有登录/注册界面，登录完之后是主界面，嗯然后中间一个开始点名按钮，点击之后跳到一个表单，填写课程名、有效时间，然后是一张二维码，结束后是一张签到表，上面有个导出为excel的按钮。

嗯，我感觉这样有点简单了，于是又脑补了首页上显示所有已签到的课程，点击课程可以跳到相关的签到表，签到表可以增删改查等。后来觉得，真的是no zuo no die，没事找事干，累死自己活该。

上面是对网站的设想，对app的设想就是扫描二维码得到对应的信息，然后访问api接口，接受返回的json数据，显示已签到的课程，相对网站来说没那么复杂。

在草稿纸上描摹了网站的大概流程后，用xmind记录了网站、app的相应的功能、流程、用什么技术实现等，之后开始用Axure画网站的原型图，app的原型图我没有画，因为对MD的组件、规范有些了解了，长什么样已经在脑海里成型了，关键是也不复杂，重点就一个主界面而已。

准备数据库

嗯，这个过程又是很痛苦的一个过程，当我设计数据库的时候我才发现自己不会！！！大一学过的数据库理论已经忘光了，什么ER图什么鬼？？？

……此处省略几百字……

在经过问童鞋，qq群里问人之后，我终于设计出来了！ssl验证，密码加密后再存储，用户权限，用户角色等等，统统不考虑!……

结果只有三张表，一张用户表，一张二维码表，一张签到表。咦？怎么会有一张二维码表？是这样的，到这步的时候，我用一个随机数来生成二维码，因为一开始的计划：课程名+时间+日期 证实不可行。

ps：到了后面的时候又将随机数换成当前timemills方便app端转换时间

哪三张表呢？

用户表：老师的工号 密码 姓名

二维码表：随机数 课程名 工号

学生表：学生的学号 姓名 随机数

当时的考虑是这样的，一个老师可以签到多次，也就是生成多个二维码，一对多的关系；一个二维码对应多个学生，一个学生对应多个二维码，多对多的关系。

行动！敲代码！

数据库有了，接下来就是敲代码了！问题又来了，我没做过网站啊……查了很多，比如python，php，ssh，甚至wordpress。最终选择还是用java吧，毕竟大学学的主要是java，一开始打算用ssh框架做网站，后来发现，其实java web就可以啊！在网上自学了一段时间后，开始把学到的知识应用到签到系统的网站端。

java web

其实网站做起来还是比较轻松的，……此处省略几百个bug……，因为原型图已经设计好了，数据库也设计好了，并且你也会技术了，接下来就是按照设计图实施而已。

主要原理就是MVC模式，M就是bean+dao类，bean类就是和数据库表对应的类，dao类就是对对应的表进行增删改查操作；V就是jsp页面，利用jstl、el标签对数据进行展示；C就是servlet，jsp里的表单的action属性的值对应着一个servlet，当提交表单的时候会跳转到对应的servlet，servlet调用dao类的方法处理数据之后，将数据在发送到jsp页面进行显示。不同的servlet之间，servlet和jsp之间可以通过内置对象比如request传递数据。

同时还有filter来过滤请求，实现点击用户登录/点击退出登录的。然后用tomcat作为服务器，jsp页面用了bootstrap，还用了其他第三方js框架。

android app

app又是一坑，因为完全靠自学的，很多东西不知道怎么做，网上的一堆框架对我来说又看不懂。首先是扫描二维码，使用ZXingLib框架，然后得到二维码信息后，发起网络请求又要命了，好不容易网络请求成功之后，又不知道怎么解析json数据，最后用了android自带的解析方法。最后MD的侧滑菜单又折腾了一阵，终于大功告成。

结语

最后，感谢您看到这里。怎么说呢，这是我自己真正意义第一个项目吧，虽然看起来不大，但其设计和实现是我自己做出来的，不是说照着网上找的教程/视频做的，你在网上搜“签到系统”是搜不到的。虽然技术是从网上学的，一开始思路也是网上的视频给我的。从一开始不知道怎么做，然后不断地尝试，踩过无数的坑，不断的完善，从一个模糊的想法，最终实现了效果！尽管它还有bug没解决，现在看来在代码优化上也有很大的改进空间，不过总算是做出来了。
